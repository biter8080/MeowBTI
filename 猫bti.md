# 产物技术与合规约束（严格遵守）
{tech_constraints: 
- 交付形态: 产物可以为两种形态之一：a) 单文件：所有代码和资源内联于一个 index.html；b) 多文件结构：index.html 通过相对路径引用本地的 js/, images/, audio/ 等子目录中的资源。 
- 纯离线运行: 整个作品必须完全离线运行，所有资源必须包含在项目目录内。 
- 无网络请求: 绝对禁止使用任何形式的网络请求 API，包括 fetch, XMLHttpRequest, axios, WebSocket 等。 
- 无外部资源: 禁止从任何外部 CDN 或服务器加载资源。如需加载配置文件，请直接写入 JS 或使用内联 JSON (<script type="application/json">) 解析。 
- 无外部跳转: 绝对禁止使用 <a> 标签、window.location 或任何其他方式跳转到外部链接。 
- 轻量化: 最终产物打包为 zip 后，总体积必须小于 8MB。 
- 数据存储: 如果需要存储数据（如最高分），推荐使用 localStorage。 
- UI 适配: 
  - 屏幕方向: {screen_orientation: 竖屏/横屏}。 
  - 自适应布局: 内容界面必须能动态、无缝地适配各种尺寸的移动端屏幕，确保在任何主流手机上都能完整显示，不能出现横向滚动条。 
- 错误处理: 必须包含 try-catch 块来捕获潜在错误，并在出错时在页面上显示友好的提示文案：{error_handling_text: 哎呀，出错了，请重启试试吧~}。 
- 合规要求: {negative_prompt: 内容不得包含任何违法违规、IP 侵权、不利于未成年人的内容。}
